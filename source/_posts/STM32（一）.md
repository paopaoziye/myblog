# 单片机
## STM32
### STM32（一）
#### 1.工程构建
**①资料下载**
>**概述**：在[keil官网](https://www.keil.arm.com/devices/)下载**芯片支持包**，在[STM32官网](https://www.st.com.cn/zh/embedded-software/stm32-standard-peripheral-libraries.html)下载**标准外设库**、**数据手册**和**参考手册**
{%list%}
芯片支持包为芯片提供设备支持和调试配置，标准外设库通过封装寄存器操作从而简化开发
{%endlist%}

{%endright%}

#### 2.启动流程
**①引言**
>**概述**：`Keil`会根据**芯片型号**填充`options->target`中`ROM`和`RAM`的**起始地址和大小**，并生成对应的**链接脚本**
{%list%}
链接脚本定义了加载区域和执行区域，前者指定程序的烧录位置，后者指定程序在运行时的存储位置
{%endlist%}
{%right%}
链接脚本规定RESET段位于ER_IROM1区域的最前端，即0x08000000起始处
{%endright%}
{%warning%}
如果执行区域和加载区域地址不同，则需要在启动或运行时将代码和数据从加载区域拷贝到执行区域
{%endwarning%}
```c
; *************************************************************
; *** Scatter-Loading Description File generated by uVision ***
; *************************************************************

LR_IROM1 0x08000000 0x00080000  {    ; 定义加载区域LR_IROM1
  ER_IROM1 0x08000000 0x00080000  {  ; 定义执行区域ER_IROM1
   *.o (RESET, +First)               ; 目标文件，其中RESET段放在这个执行区域的最前面
   *(InRoot$$Sections)               ; ARM库需要的特殊段
   .ANY (+RO)                        ; 只读段，通常是代码段和常量
   .ANY (+XO)                        ; 可执行的只读段
  }
  RW_IRAM1 0x20000000 0x00020000  {  ; 定义执行区域RW_IRAM1
   .ANY (+RW +ZI)                    ; 可读写和零初始化的数据段
  }
}
```
**②启动文件**
>**概述**：为**堆栈**划分地址空间，并定义了**中断向量表**、**复位处理函数**和**各个中断的默认服务函数**
{%list%}
通常情况下STM32将0x08000000映射到0x00000000，但是可以通过设置BOOT引脚修改该逻辑实现不同启动方式
{%endlist%}
{%right%}
STM32上电后，会从地址0x00000000处读取32位值赋给MSP，并从0x00000004处读取32位值赋给PC
{%endright%}
>综上所属，**系统上电**后会将`MSP`设置为`__initial_sp`并跳转到`Reset_Handler`执行
{%warning%}
所有中断服务函数默认为死循环，且定义为[WEAK]，可以在C代码中重写
{%endwarning%}
```nasm
;栈大小为1KB，为栈分配空间并将__initial_sp设置为栈顶指针
Stack_Size      EQU     0x00000400

                AREA    STACK, NOINIT, READWRITE, ALIGN=3
Stack_Mem       SPACE   Stack_Size
__initial_sp
;定义堆大小为512B，并定义堆的起始地址__heap_base和结束地址__heap_limit
Heap_Size       EQU     0x00000200

                AREA    HEAP, NOINIT, READWRITE, ALIGN=3
__heap_base
Heap_Mem        SPACE   Heap_Size
__heap_limit

                PRESERVE8
                THUMB
; 中断向量表
                AREA    RESET, DATA, READONLY
                EXPORT  __Vectors
                EXPORT  __Vectors_End
                EXPORT  __Vectors_Size

__Vectors       DCD     __initial_sp               ; 初始栈顶地址
                DCD     Reset_Handler              ; 复位处理函数地址
                ;...其余略
__Vectors_End 
;各个中断默认服务函数，略           
```
**③`Reset_Handler`**
>**概述**：调用`SystemInit`进行`FPU`和**系统时钟**的配置，最后跳转到`__main`
{%list%}
初始时采用内部高速时钟，在SetSysClock中启用外部高速时钟，并配置PLL和总线分频器完成系统时钟的初始化
{%endlist%}
{%right%}
__main将FLASH中已初始化的全局变量拷贝到RAM中并清零BSS段，最后跳转到用户定义的main函数
{%endright%}
{%warning%}
初始时需要禁用外部高速时钟相关以及时钟中断
{%endwarning%}
```nasm
Reset_Handler    PROC
                 EXPORT  Reset_Handler             [WEAK]
        IMPORT  SystemInit
        IMPORT  __main

                 LDR     R0, =SystemInit
                 BLX     R0
                 LDR     R0, =__main
                 BX      R0
                 ENDP
```
```c
void SystemInit(void)
{
  //如果芯片支持FPU且启用了FPU，则允许全权限访问浮点单元
  #if (__FPU_PRESENT == 1) && (__FPU_USED == 1)
    SCB->CPACR |= ((3UL << 10*2)|(3UL << 11*2)); 
  #endif
  //启用内部高速时钟
  RCC->CR |= (uint32_t)0x00000001;
  //复位外部时钟配置寄存器
  RCC->CFGR = 0x00000000;
  //禁用外部高速时钟、时钟安全系统和锁相环
  RCC->CR &= (uint32_t)0xFEF6FFFF;
  //复位PLL配置寄存器
  RCC->PLLCFGR = 0x24003010;
  //禁用外部晶振的旁路模式
  RCC->CR &= (uint32_t)0xFFFBFFFF;
  //关闭所有时钟中断
  RCC->CIR = 0x00000000;
  //如果使用了外部SRAM或SDRAM，则初始化对应的存储器控制器
  #if defined(DATA_IN_ExtSRAM) || defined(DATA_IN_ExtSDRAM)
    SystemInit_ExtMemCtl(); 
  #endif 
  //系统时钟配置
  SetSysClock();
  //如果在SRAM定义了中断向量表，则进行重定位
  #ifdef VECT_TAB_SRAM
    SCB->VTOR = SRAM_BASE | VECT_TAB_OFFSET; 
  #else
    SCB->VTOR = FLASH_BASE | VECT_TAB_OFFSET; 
  #endif
}
```
#### 3.系统时钟
**①时钟源**
>**概述**：用于提供**时钟信号**，`CPU`以及**各种外设**的操作需要时钟信号同步，如下所示
{%list%}
时钟源可分为高速内部时钟HSI、高速外部时钟HSE、低速内部时钟LSI和低速外部时钟LSE
{%endlist%}
{%right%}
外部时钟的时钟信号由外部晶振产生，精度高且稳定性好，适用于高精度场景
{%endright%}
{%warning%}
内部时钟的时钟信号由芯片内部的RC振荡器产生，精度和稳定性较低，主要用于启动外部时钟或低精度场合
{%endwarning%}
![时钟源](/image/stm32_17.png)
**②时钟树**
>**概述**：不同外设使用的**时钟源**和**频率**不同，需要使用**分频器**、**锁相环**和**复用器**进行配置，如下所示
{%list%}
初始时使用HSI为时钟源，且AHB、APB1和APB2的分频器系数均为/1
{%endlist%}
{%right%}
SetSysClock将时钟源从HSI修改为经过锁相环乘以九HSE，且AHB、APB1和APB2的分频器系数为/1、/1和/2
{%endright%}
{%warning%}
部分片上外设内部有自己的分频器，需要在初始化时进行设置
{%endwarning%}
![时钟树](/image/stm32_18.png)

#### 4.GPIO
**①简介**
>**概述**：称为**通用输入输出端口**，可以被配置为**输入模式**或**输出模式**，用于与**外部设备**进行简单的**数字信号交互**
{%list%}
STM32将GPIO引脚分为多个组，每一组GPIO引脚由外设GPIOX控制，如GPIOA、GPIOB和GPIOC等
{%endlist%}
{%right%}
每个GPIOX都有一组寄存器，可以通过读写相关寄存器控制该组GPIO引脚，如GPIOA1、GPIOA2和GPIOA3等
{%endright%}
{%warning%}
一个寄存器可以控制多个GPIO引脚，所以在修改一个引脚配置时不能影响其他引脚
{%endwarning%}
```shell
[时钟/电源]
|
├──[GPIOA]——[GPIOAx]
|
├──[GPIOB]——[GPIOBx]
|
[...]
```
**②输入模式**
>**概述**：当`GPIO`引脚被设置为**输入模式**时，可以用于**读取外部数字信号**，其电路简图如下所示
{%list%}
可分为浮空输入、上拉输入、下拉输入和模拟输入，其中模拟输入模式用于读取模拟信号，暂不介绍
{%endlist%}
{%right%}
模拟输入模式下输入信号不会经过施密特触发器，其余三种模式的输入信号会被施密特触发器转化为高低电平
{%endright%}
{%warning%}
当输入信号较弱，容易受到干扰时，选择上拉输入模式或下拉输入模式
{%endwarning%}
>**浮空输入**：不使用**上拉/下拉电阻**，当引脚为**高电平/低电平**时，输入值为`1/0`，**无输入**时输入值**不稳定**

>**上拉输入**：使用**上拉电阻**，当引脚为**高电平/低电平**时，输入值为`1/0`，**无输入**时输入值为`1`

>**下拉输入**：使用**下拉电阻**，当引脚为**高电平/低电平**时，输入值为`1/0`，**无输入**时输入值为`0`
```shell
                         [VDD] -> 3.3v
                           | 
                          [R]  -> 上拉电阻
                           |
[输入寄存器]----------------+----------------[GPIO引脚]
                           | 
                          [R]  -> 下拉电阻
                           |
                         [VSS]
```
**③输出模式**
>**概述**：当`GPIO`引脚被设置为**输出模式**时，可以用于**向外输出数字信号**，其电路简图如下所示
{%list%}
可分为通用推挽输出、通用开漏输出、推挽复用输出和开漏复用输出，主要是通用和复用以及推挽和开漏的区别
{%endlist%}
>**通用**表示**输出信号**直接来自于`CPU`，**复用**表示**输出信号**来自于**其他片上外设**
{%right%}
推挽输出模式可以将引脚视作为一个电源，开漏输出模式可以将引脚视作为一个开关，需要外接电源才能正常工作
{%endright%}
{%warning%}
输出速度通常选择满足要求的最小值，因为信号变化需要一定的上升/下降时间，输出速度太快会导致保持时间过短
{%endwarning%}
>**推挽输出**：使用`P-MOS`和`N-MOS`，当输出为`1/0`时，`P-MOS/N-MOS`导通，引脚为**高电平/低电平**

>**开漏输出**：只使用`N-MOS`，当输出为`1/0`时，`N-MOS`**阻塞/导通**，引脚为**高阻态/低电平**

```shell
                            [VDD]                       [外部VDD] ->开漏模式所需外部电源
                              |                             |
                           [P-MOS]                         [R]    ->上拉电阻
                              |                             |    
[输出寄存器]----[输出控制器]----+----------------[GPIO引脚]----
                              | 
                           [N-MOS]
                              |
                            [VSS]
```
