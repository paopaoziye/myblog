ARM体系结构

1.引言

1.1简介

①处理器系列

概述：常见的处理器系列有Cortex-M和Cortex-A系列，分别针对不同的使用场景，具有不同的特性

Cortex-M主要面向低功耗场景，使用ARM-M指令集架构，Cortex-A主要面向高性能场景，使用ARM-A指令集架构

Cortex-M只有简单的内存管理，且只支持单核，Cortex-A支持MMU、虚拟内存管理和多核

Cortex-M对中断管理进行了优化，更适合实时应用

②指令集

概述：ARM体系架构提供ARM指令集和Thumb指令集，前者指令长度为32位，后者指令长度为16位或32位

Thumb指令集提供了更出色的代码密度，但是通常运行速度不如ARM指令集
某些处理器系列提供Thumb-2指令集，可以混合使用ARM指令集和Thumb指令集

③指令集切换

概述：通常采用BX addr/addr+1表示后续使用ARM/Thumb指令集

ARM处理器在处理异常时，不管处理器处于什么状态，则都将切换到ARM状态

在汇编文件中，可以采用.arm/.thumb表示后续指令使用ARM/Thumb指令集

使用GCC时，可以添加-marm/-mthumb编译选项表示使用ARM/Thumb指令集编译该文件

ARM指令和Thumb指令的位0始终为0，所以采用位0来表示ARM指令与Thumb指令的切换标志位
指令集的切换本质上是修改了程序状态寄存器对应位，但是不要直接修改对应位进行指令集切换

1.2Cortex-M系列

①处理器状态

概述：有异常模式和线程模式两种运行模式，前者一直处于特权级，后者分为特权级线程模式和用户级线程模式

系统上电复位时处于特权级线程模式，通过手动修改CONTROL寄存器降级为用户级线程模式

CONTROL寄存器用于定义特权等级和堆栈指针的选择

用户级线程模式不能修改CONTROL寄存器，需要触发异常中断，在中断中修改CONTROL寄存器来实现等级提升


②内部寄存器

概述：分为通用寄存器R0-R12、堆栈寄存器SP、链接寄存器LR、程序计数器PC和程序状态寄存器xPSR等

处理状态不同，SP分别指向主堆栈或线程堆栈，前者用于操作系统内核以及异常处理例程，后者用于应用程序代码

LR在调用函数时保存函数的返回地址，在中断服务程序中用作EXC_RETURN,用于确定中断结束后恢复状态

PC的最低位并不是地址，而是Thumb状态位，会被写到到EPSR的T位

除此之外，还有一些只能在特权级访问的寄存器CONTROL、BASEPRI、PRIMASK和FAULTMASK

这些寄存器用于设置特权等级和屏蔽中断等

程序状态寄存器xPSR在用户级只能被访问，不能被修改

③过程调用标准

概述：调用函数/中断时，R0-R3用于传递参数，R4-R11则用于保存临时变量，R0用于保存返回值

若传递参数多于四个，多余的参数用栈传递，若返回值为64位，R1也用来保存返回值
调用者需要对

函数或子程序应该保持R4-R11、R13和R14的数值。若这些寄存器在函数或子程序执行期间被修改，则其函数应该保持在栈中并在返回调用代码前恢复。这几个寄存器也被称作“被调用者保存寄存器”，也就是需要被调用者（例如子函数，中断等）进行保存的寄存器。而对于R0-R3、R12、则属于调用者保存寄存器，这几个寄存器是需要调用者做保存工作。在发生异常或中断啊时，R0-R3、R12、SP、PC会硬件自动进程压栈。

1.3Cortex-A系列

①处理器状态

概述：有用户模式、系统模式、管理员模式、终止模式、未定义指令模式、一般中断模式和快中断模式

用户模式处于用户态，其余模式处于特权态，用户态需要通过中断/异常切换到特权态，特权态之间可以相互切换
类似的，Cortex-A通过修改CPSR的MODE域来进入其他模式

②内部寄存器

概述：类别同Cortex-M，但是每种特权模式都有自己的某些备份寄存器，会有对应的后缀

异常模式有自己的SP、LR和SPSR寄存器，SP指向自己的栈，LR/SPSR保存异常处理完毕后的返回地址/程序状态
每种异常模式只能访问用户态的寄存器以及对应的备份寄存器

2.常用指令

2.1引言

①一般语法格式

概述：一般语法格式为<指令助记符>{<执行条件>}{S} <目标寄存器>,<操作数>{,<第二个操作数>}

<>内的项是必须的，{}内的项是可选的，S表示该指令会影响到程序状态寄存器的条件标志位
执行条件根据程序状态寄存器的条件标志位决定是否执行该指令
条件码 含义 CPSR 标志位要求
EQ 等于 Z = 1
NE 不等于 Z = 0
CS 进位设置 C = 1
CC 进位清除 C = 0
MI 负数 N = 1
PL 正数 N = 0
VS 溢出设置 V = 1
VC 溢出清除 V = 0
HI 高于 C = 1 且 Z = 0
LS 低于或相等 C = 0 或 Z = 1
GE 大于或等于 N = V
LT 小于 N ≠ V
GT 大于 Z = 0 且 N = V
LE 小于或等于 Z = 1 或 N ≠ V
AL 总是 无条件（默认）

设置了链接地址，程序会人为数据和指令在对应位置，如果将程序拷贝到了错误的位置，则链接地址没有数据和指令，会导致错误

代码段、只读数据段和可读可写数据段，如果其不在链接地址就需要重定位
BSS不需要重定位，因为程序不保存BSS段，使用前将BSS段对应空间清零即可

中断源通过中断控制器发送信号给CPU
指令不对、数据访问等即异常会打断CPU
GPIO通常会先发送中断给外部中断控制器，外部中断控制器再发送给中断控制器

CPU模式
用户
特权模式：系统和异常，特权模式可以相互切换（操作CPSR），用户模式不能直接切换到特权模式

特权模式不仅可以访问用户状态的寄存器，还有自己的专属寄存器

中断

GPIO控制器选择对应GPIO引脚作为外部中断0、1…，即让对应引脚可以发出中断，当然也可能没有相关设置，全部由外部中断控制器实现
外部中断控制器控制中断源的触发方式以及是否屏蔽中断等，配置对应寄存器允许终端以及触发方式
NVIC设置对应寄存器，开启对应中断，使其可以将中断发给CPU，选择最高优先级的中断发送给CPU
